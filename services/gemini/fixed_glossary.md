正在做日綜／日本搞笑藝人字幕的日文→台灣繁中翻譯。目標不是正式學術翻譯，而是讓中文字幕自然、好讀、辨識度高，同時不要亂創台式譯名。

## 檔案結構（talents / others）

`fixed_glossary.json` 是一個物件，分兩大類：

```jsonc
MappingBlock = { "jp": [str, ...], "zh": str }
{
  "talents": [
    { "group": MappingBlock,            // 可選：組合名；solo 藝人省略此鍵
      "members": [MappingBlock, ...] }   // 至少一名成員
  ],
  "others": [ MappingBlock, ... ]        // 節目／單元／品牌／術語／作品 等非人物
}
```

- **talents**：搞笑藝人。組合用一個 unit 包起來：`group` 放組合名，`members` 放成員。單人藝人省略 `group`，只有 `members`。
- **others**：節目名、單元名、品牌、平台、搞笑術語、作品名等非人物專名。

為什麼要把組合和成員包在一起：很多裸成員名（`きむ`／`リリー`／`ガク`／`ノブ`／`せいや`）單獨看極度模糊。把成員掛在組合底下後，比對命中任一成員或組合名時，整個 unit（組合＋全部成員）會一起餵給 pre-pass，模糊成員名就有組合當辨識上下文。

## jp[] 使用原則
`jp[]` 只放組合／人物**最原始、正確的 source 來源拼寫**。

- NFKC（全形→半形、相容字）
- 片假名 → 平假名
- 去除長音符 `ー`、波浪 `〜～`、中點 `・･`、空白、Tab、換行、`!！?？.。,、:：/／「」『』()（）`
- casefold（大小寫）

凡是只差這些的變體，留在 jp[] 是死重複，**不要收**。多個 jp 元素只在「確實存在多種正規來源寫法」時才保留（是不同正寫，不是變體）。

ASR 容錯責任明確下放：

1. `_normalize_jp` 處理可折疊漂移（假名互換、全半形、空白、長音、標點、大小寫）。
2. full 模式 + LLM 處理相似音／嚴重走鐘（`FIXED_GLOSSARY_FULL_INSTRUCTION` 已明載容許 ASR 誤聽，並以音訊／畫面確認）。

好例：

```jsonc
{ "jp": ["熊元プロレス"], "zh": "熊元摔角" }   // 不收 "熊元 プロレス"/"熊元　プロレス"，normalize 已折
{ "jp": ["ほんこん"], "zh": "Honkon" }          // 不收 "ホンコン"，片→平假名已折
```

壞例（不要這樣）：

```jsonc
{ "jp": ["ほんこん", "ホンコン", "ホンコ"], "zh": "Honkon" }   // 前二者 normalize 同；第三是相似音猜測
{ "jp": ["ホンコン", "香港"], "zh": "Honkon" }                  // 香港 漢字出現時很可能真的是地名
```

## 專名翻譯總原則

專名包含：節目名、單元名、搞笑團體名、藝人名、藝名、作品名、品牌名、梗名。

整體優先順序（命中第一個適用者即停）：

1. **已有穩定台灣／中文譯名就用既有譯名**
   - 只在能確認常見、穩定、官方或中文圈已廣泛使用時採用。
   - 不要自己發明看起來很像台灣節目名的「創意譯名」。
   - 例：`ロンドンハーツ`／`ロンハー` → `男女糾察隊`；`逃走中` → `全員逃走中`；`るろ剣` → `神劍闖江湖`；`南海キャンディーズ` → `南海甜心`；`ネプチューン` → `海王星`

2. **人名／藝名可回推漢字時，要看該漢字是不是名稱核心**
   - 假名只是本名／姓氏的假名表記，或中文圈已有穩定漢字用法，可用繁中漢字：`河井ゆずる`／本名 `河井譲` → `河井讓`；`お見送り芸人しんいち`／本名 `上野晋一` → `送別藝人晉一`；`イワクラ`／`岩倉美里` → `岩倉`；`月亭方正` → `月亭方正`
   - 假名部分可明確回推姓名漢字、且該漢字是藝名來源／梗點時保留漢字，其餘外來語／假名再用英文或羅馬字：`きむらバンド` → `木村Band`（不建議 `Kimura Band`／`木村樂團`）
   - 藝名刻意片假名化／暱稱化／角色化時，即使知道本名漢字也不一定回推，通常保留藝名讀音用羅馬字：`ユースケ` → `Yusuke`（不必 `裕介`）；`ガク` → `Gaku`；`ケツ` → `Ketsu`；`きょん` → `Kyon`（不擅補 `恭`）；`おたけ` → `Otake`（不擅補 `小武`/`阿武`）

3. **語意清楚、翻成中文後仍像專名時，才翻中文**
   - 可翻：`熊元プロレス` → `熊元摔角`；`紅しょうが` → `紅生姜`；`ママタルト` → `媽媽塔`；`チャンスの時間` → `機會的時間`；`囲碁将棋` → `圍棋將棋`；`いろはに千鳥` → `伊呂波千鳥`
   - 不建議直譯：`見取り図` → `Mitorizu`（非 `示意圖`）；`銀シャリ` → `銀Shari`（非 `銀舍利`）；`モグライダー` → `Mogu Rider`（非 `鼴鼠騎士`）；`おいでやす小田` → `Oideyasu小田`（非 `歡迎光臨小田`）；`ロングコートダディ` → `Long Coat Daddy`；`カナメストーン` → `Kaname Stone`（非 `要石`）；`サルゴリラ` → `Sarugorilla`；`アンタッチャブル` → `Untouchable`

4. **純假名／片假名藝名、團名，沒有可回推漢字或清楚語意時，優先羅馬字**
   - 中文字幕裡不要隨便保留日文假名，除非保留假名真的比較有辨識度。
   - 漢字＋假名混合表記，通常保留有辨識度的漢字，假名部分羅馬字化；若已有官方／常見英文表記，或保留漢字會像自創混搭名，則整體羅馬字。
   - 例：`松井ケムリ` → `松井Kemuri`；`真中まな` → `真中Mana`；`鈴木もぐら` → `鈴木Mogura`；`今井らいぱち` → `今井Raipachi`；`銀シャリ` → `銀Shari`；`や団` → `Yadan`；`なえなの` → `naenano`；`ほんこん` → `Honkon`；`べーやん` → `Beeyan`

5. **羅馬字優先用官方／常見英文表記，不要機械逐音轉寫**
   - 已有官方／常見／資料庫慣用英文表記時優先採用，固定大小寫與空格，組合名與成員一致。
   - 例：`ダウ90000` → `Daw90000`；`かもめんたる` → `Kamomental`；`パンプキンポテトフライ` → `Pumpkin Potato Fry`；`ダウンタウン` → `DOWNTOWN`；`ダウンタウンDX` → `DOWNTOWN DX`；`水曜日のダウンタウン` → `星期三的DOWNTOWN`；`さまぁ〜ず` → `SUMMERS`

6. **保留日文原文（尤其假名）是最後手段**
   - 只有翻中文和羅馬字都會降低辨識度，或是明顯日文文字遊戲／節目標題時才保留。
   - 例：`かまいガチ` 可保留 `かまいガチ`。
   - 不要預設保留成假名的：`カカロニ`／`ケムリ`／`すがちゃん`／`や団`／`なえなの`／`ほんこん`。

## 專名翻譯和一般字幕翻譯要分開

一般字幕句子中，不要擅自補「我／你／他／她／我們／大家」等日文省略主詞。

但節目名、藝名、梗名是固定名詞，可以為了保留語氣做固定化翻譯。例：一般句子 `知らんけど` 不一定補成「我是不知道啦」；但節目名 `かまいたちの知らんけど` 可翻 `鎌鼬的我是不知道啦`。

## 不要做的事

- 不要把片假名藝名直接照意思硬翻。壞：`松井ケムリ` → `松井煙`；好：`松井Kemuri`
- 不要擅自幫假名人名補漢字。壞：`きょん` → `恭`；好：`Kyon`
- 不要把節目名翻成很像台灣節目的創意標題。壞：`かまいガチ` → `鐮鼬真挑戰`；好：`かまいガチ`
- 不要把搞笑團體名硬翻成普通名詞。壞：`見取り図` → `示意圖`；好：`Mitorizu`
- 不要把字面有意思的藝名翻成普通句子。壞：`おいでやす小田` → `歡迎光臨小田`；好：`Oideyasu小田`
- 不要把暱稱化藝名翻成自創中文暱稱。壞：`おたけ` → `小武`/`阿武`；好：`Otake`
- 不要在中文字幕裡大量留下日文假名。翻成中文後仍像專名才中文化；否則優先羅馬字／常見英文表記；保留假名放最後。
- 不要把「組合＋成員」拼成一條 jp 別名；組合關係用 talents 結構表達。
